# Copilot instructions for this project

This file tells GitHub Copilot how it should behave when helping with code in this repository.

## 1) Purpose

> Explain the intent of this repo and what kinds of tasks Copilot should help with.

- This repo contains educational programming assignments (Python + web). Copilot should help with examples, explanations, and completing starter code without breaking existing assignment scaffolding.

## 2) Scope (where this applies)

- Apply these guidelines to any source file under:
  - `assignments/`
  - `assets/js/`
  - `assets/css/`
  - `index.html`, `config.json`, any root-level files

## 3) Conventions (what to enforce)

### Python
- Target Python 3.11+ (or the version used by the environment).
- Use **4 spaces** for indentation, no tabs.
- Follow **PEP 8** for formatting (e.g., snake_case for functions/variables, PascalCase for classes).
- Do not modify starter code outside clearly marked "TODO" blocks.

### JavaScript / HTML / CSS
- Prefer modern JavaScript (ES6+) syntax (`const`/`let`, arrow functions, template literals).
- Keep HTML semantic and accessible.
- Keep CSS styles scoped and avoid global styles when possible.

## 4) When in doubt / clarifications

- If a requirement is ambiguous, ask the user for clarification instead of guessing.
- When asked to write new code for an assignment, keep changes confined to the relevant file and follow the assignment instructions.

## 5) Example prompts to use (for contributors)

- "Implement the function in `assignments/python-basics/starter-code.py` to solve the exercise described in the README."
- "Add a new helper function to `assets/js/script.js` that validates the form on `index.html`."
